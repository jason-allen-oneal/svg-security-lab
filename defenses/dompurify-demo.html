DOMPurify SVG Sanitization

✅ DOMPurify Active: DOMPurify library is loaded and ready to sanitize SVG content

DOMPurify is a DOM-only, super-fast, uber-tolerant XSS sanitizer for HTML, MathML and SVG. It's written in JavaScript and works in all modern browsers.

How DOMPurify Works:

  • Parses: Converts HTML/SVG strings to DOM nodes
  • Sanitizes: Removes dangerous elements and attributes
  • Serializes: Converts back to safe HTML/SVG string

DOMPurify Configuration Examples

1. Basic SVG Sanitization

// Basic sanitization - removes all scripts const cleanSVG = DOMPurify.sanitize(svgContent);

2. Custom SVG Configuration

// Allow only specific SVG elements and attributes const cleanSVG = DOMPurify.sanitize(svgContent, { ALLOWED_TAGS: ['svg', 'rect', 'circle', 'text', 'path'], ALLOWED_ATTR: ['width', 'height', 'fill', 'stroke', 'cx', 'cy', 'r'], FORBID_TAGS: ['script', 'foreignObject'], FORBID_ATTR: ['onload', 'onclick', 'onerror'] });

3. Strict SVG Configuration

// Very restrictive configuration const cleanSVG = DOMPurify.sanitize(svgContent, { ALLOWED_TAGS: ['svg', 'rect', 'circle'], ALLOWED_ATTR: ['width', 'height', 'fill'], KEEP_CONTENT: false, RETURN_DOM: false, RETURN_DOM_FRAGMENT: false });

4. Sanitization with Custom Hooks

// Add custom sanitization logic DOMPurify.addHook('beforeSanitizeElements', function(current, hook, node) { // Remove any element with 'on' attributes if (node.hasAttribute && node.hasAttribute('on')) { return 'REMOVE_ELEMENT'; } return current; }); const cleanSVG = DOMPurify.sanitize(svgContent);

Real Attack Examples

Attack Vector 1: Script Injection

// Malicious SVG with script <svg> <script>alert('XSS')</script> <rect width="100" height="100" fill="red"/> </svg> // After DOMPurify sanitization <svg> <rect width="100" height="100" fill="red"/> </svg>

Attack Vector 2: Event Handlers

// Malicious SVG with event handlers <svg> <rect width="100" height="100" fill="red" onload="alert('XSS')" onclick="alert('XSS')"/> </svg> // After DOMPurify sanitization <svg> <rect width="100" height="100" fill="red"/> </svg>

Attack Vector 3: Foreign Objects

// Malicious SVG with foreignObject <svg> <foreignObject> <script>alert('XSS')</script> </foreignObject> </svg> // After DOMPurify sanitization (with FORBID_TAGS) <svg></svg>

Implementation Best Practices

  • Always Sanitize: Never trust user-provided SVG content
  • Use Strict Configurations: Only allow necessary elements and attributes
  • Test Thoroughly: Verify sanitization works against known attack vectors
  • Keep Updated: Use the latest version of DOMPurify
  • Combine Defenses: Use DOMPurify with CSP and other protections
